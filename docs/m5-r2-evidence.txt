==============================================================================
INTERACTIVE FLOWS (described, not captured)
==============================================================================
The `monkeydo` IPC client doesn't programmatically drive touch events,
so the following flows are not captured as stdout — they're documented
here as the expected behavior wired by the M5 patch and verified via
unit tests + code review:

  1. Type "ש" → diagnostic fires `M5 rank: buf='ש' top=[שלום,שבת]`,
     suggestion area renders 2 lines (שלום first by popularity 100 > 60).
  2. Type "ב" (buffer now "שב") → `M5 rank: buf='שב' top=[שבת]`,
     suggestion area renders 1 line.
  3. BACKSPACE the "ב" → buffer back to "ש", top suggestions back to 2.
  4. BACKSPACE the "ש" → buffer empty, top suggestions back to 3.
  5. Tap the "שלום" suggestion line → `ArticleStore.bodyOf("shalom")`
     returns `Strings.sampleArticle()` (~2058 bytes), wikiwatchView is
     constructed with that body, pushed onto the view stack via
     `WatchUi.pushView(reader, readerDelegate, WatchUi.SLIDE_LEFT)`.
     The M2.x reader displays the long Hebrew article. Drag-scroll
     works as in M2.x (live `onDrag` per M2.1, double-tap-edge
     scroll-to-endpoints per M2.5).
  6. Press lower-right physical back button → reader pops, returns to
     the keyboard with the typed buffer still in place.

The code paths exercised by 1–6 are all reachable from `onTap` in
`source/wikiwatchKeyboardDelegate.mc` and route through:
  - SPACE / BACKSPACE / sub-button tap → _recomputeSuggestions() →
    _view.setSuggestions(...)
  - In-band suggestion tap (x within band, y within line row) →
    _view.suggestionAt(x, y) → non-null → WatchUi.pushView(reader)
